Comparación Detallada

ATDF ha evolucionado desde su versión inicial 0.1.0 a la versión mejorada 0.2.0, manteniendo la compatibilidad completa con las versiones anteriores mientras añade características avanzadas.

Tabla Comparativa

Característica Version 0.1.0 (Básica) Version 0.2.0 (Mejorada)
Estructura básica ✓ Estructura básica con campos esenciales ✓ Misma estructura básica con campos adicionales opcionales
Campos requeridos ✓ tool_id, description, when_to_use, how_to_use ✓ Mismos campos requeridos
Metadatos ✗ No soportado ✓ Campo opcional metadata con versión, autor, etiquetas, etc.
Soporte multilingüe ⚠ Limitado (archivos separados para cada idioma) ✓ Integrado a través del campo localization
Prerrequisitos ✗ No soportado ✓ Campo opcional prerequisites
Mecanismos de retroalimentación ✗ No soportado ✓ Campo opcional feedback con señales de progreso y finalización
Ejemplos de uso ✗ No soportado ✓ Campo opcional examples con ejemplos completos
Tipos de entrada complejos ⚠ Limitado (solo tipos básicos) ✓ Esquemas de objetos complejos con propiedades anidadas
Compatibilidad hacia atrás ✓ N/A (primera versión) ✓ Totalmente compatible con 0.1.0
Validación de esquema ✓ Validación básica ✓ Validación mejorada con campos adicionales
Selección de herramientas ✓ Coincidencia básica por descripción y uso ✓ Coincidencia mejorada con conciencia de contexto
Detección de idioma ✗ No soportado ✓ Detección automática de idioma

Mejoras Principales en la Versión 0.2.0

Organización Mejorada

La adición de campos de metadatos facilita la organización y categorización de herramientas en colecciones más grandes.

Mejor Soporte Multilingüe

En lugar de requerir archivos separados para cada idioma, la versión 0.2.0 permite definir múltiples variantes de idioma dentro de un solo archivo.

Mejoras de Seguridad

Los prerrequisitos permiten a los agentes verificar si se cumplen las condiciones necesarias antes de usar una herramienta.

Mejor Aprendizaje

Los ejemplos de uso proporcionan escenarios concretos que ayudan a los agentes a entender cómo usar las herramientas de manera efectiva.

Monitoreo de Progreso

Los mecanismos de retroalimentación permiten a los agentes rastrear el progreso y determinar cuándo se completan las operaciones.

Parámetros Complejos

El soporte para esquemas de objetos anidados permite interfaces de herramientas más sofisticadas.

Guía de Migración

Para actualizar de v0.1.0 a v0.2.0:

  1. Mantén todos los campos requeridos existentes de v0.1.0.
  2. Añade campos mejorados opcionales según sea necesario:
    • metadata para información organizativa
    • localization para soporte multilingüe
    • prerequisites para información de dependencias
    • feedback para monitoreo de progreso
    • examples para ejemplos de uso
  3. Considera mejorar los parámetros de entrada con esquemas complejos cuando sea apropiado.

El SDK proporciona conversión automática a través de la función convert_to_enhanced():

from tools.converter import convert_to_enhanced, load_tool, save_tool

# Cargar una herramienta básica
basic_tool = load_tool("path/to/basic_tool.json")

# Convertir al formato mejorado
enhanced_tool = convert_to_enhanced(
    basic_tool,
    author="Tu Nombre",
    extract_language=True
)

# Guardar la herramienta mejorada
save_tool(enhanced_tool, "path/to/enhanced_tool.json")
